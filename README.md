# Timary — электронный дневник школы

Проект работает как простой электронный дневник: ученики видят только свой класс, учителя — только те классы, доступ к которым выдал директор, директор управляет классами, расписанием, новостями и доступами.

## Что добавлено в этой версии

- ученик может сдавать домашнее задание файлом;
- учитель может открыть раздел «Проверка ДЗ», скачать файл ученика, принять работу или вернуть на доработку;
- директор больше не может создавать домашние задания;
- директор может удалять новости;
- директор может добавлять новые классы без работы с базой данных;
- при регистрации ученика можно выбрать класс;
- новый зарегистрированный ученик сразу появляется у директора в списке учеников;
- учитель видит нового ученика только после того, как директор выдал учителю доступ к классу ученика;
- расписание ограничено по ролям: ученик видит только свой класс, учитель — только доступные классы;
- главный экран входа упрощен, без технических упоминаний архитектуры системы;
- при регистрации добавлена проверка сложности пароля;
- добавлена отдельная схема базы данных `database_diagram.png` и исходник `database_diagram.svg`.

## Требования

1. Python 3.10 или новее.
2. PostgreSQL.
3. Созданная база данных `timary_db`.

## Создание базы данных

Откройте pgAdmin или SQL Shell и выполните:

```sql
CREATE DATABASE timary_db;
```

Таблицы вручную создавать не обязательно: приложение создаст их при первом запуске. Файл `database_schema.sql` добавлен в проект как SQL-схема базы данных для курсовой работы. Отдельная визуальная схема находится в файлах `database_diagram.png` и `database_diagram.svg`.

## Настройка подключения

В корне проекта есть файл `.env.example`. Скопируйте его и назовите копию `.env`.

В `.env` должна быть такая строка:

```env
DATABASE_URL=postgresql+pg8000://postgres:12345678@localhost:5432/timary_db
SECRET_KEY=change-me
FLASK_DEBUG=1
```

Если у PostgreSQL другой пароль, поменяйте только пароль `12345678` на свой.

## Запуск проекта

Откройте терминал в папке проекта и выполните:

```bash
python -m venv venv
venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python run.py
```

После запуска откройте в браузере:

```text
http://127.0.0.1:5000
```

## Как работать с системой

### 1. Регистрация пользователя

На экране входа нажмите «Регистрация». Пароль должен содержать минимум 8 символов, заглавную и строчную букву, цифру и специальный символ. Для ученика обязательно выберите класс. После регистрации ученик привязан к выбранному классу и будет видеть только свое расписание, свои оценки, посещаемость и домашние задания.

Для учителя при регистрации указывается должность. После регистрации учитель еще не получает доступ к классам автоматически. Это сделано специально: доступ к классам выдает директор.

### 2. Добавление класса директором

Директор открывает раздел «Классы и доступ» и использует форму «Добавить класс». После создания класс появляется в списках регистрации, расписания и выдачи доступа.

### 3. Выдача доступа учителю

Директор открывает «Классы и доступ», выбирает учителя и класс, затем нажимает «Выдать». После этого учитель увидит этот класс в разделе «Мои классы», а ученики этого класса появятся у него в журнале оценок и посещаемости.

Если доступ удалить, учитель перестанет видеть этот класс и учеников этого класса.

### 4. Расписание

Директор создает расписание в разделе «Расписание». Ученик видит только расписание своего класса. Учитель видит только расписание классов, к которым ему выдан доступ.

### 5. Домашние задания

Домашние задания создает только учитель. Директор не имеет формы создания домашнего задания.

Учитель выбирает доступный ему класс, предмет, дату сдачи, тему и текст задания. После создания задание появляется у учеников выбранного класса.

### 6. Сдача файла учеником

Ученик открывает «Домашние задания», выбирает файл в карточке задания, при необходимости пишет комментарий и нажимает «Отправить файл». Файл сохраняется в папку `uploads` внутри проекта, а учитель видит отправку в разделе «Проверка ДЗ».

### 7. Проверка работы учителем

Учитель открывает «Проверка ДЗ», скачивает файл ученика, выбирает статус «Принято» или «Вернуть на доработку», добавляет комментарий и сохраняет результат.

Учитель видит только отправки по своим заданиям и только по тем классам, к которым ему выдан доступ.

### 8. Новости

Директор может добавлять и удалять новости. Все пользователи видят новости на главной странице и в разделе «Новости».

## Важно при обновлении старой версии

Если вы запускаете эту версию поверх старой базы `timary_db`, приложение само добавит недостающие поля для отправки файлов. Но если старая база сильно испорчена тестами, проще удалить базу и создать ее заново:

```sql
DROP DATABASE timary_db;
CREATE DATABASE timary_db;
```

После этого снова запустите `python run.py`.
